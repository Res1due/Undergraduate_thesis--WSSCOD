## 1. 项目整体流程概述

本项目采用两阶段弱监督伪装目标检测框架，整体流程如下：

1. 使用少量像素级标注样本和框标注提示训练辅助网络 `ANet`。
2. 训练完成后，使用 `ANet` 为大量仅有框标注的样本生成伪标签。
3. 在本人改进版本中，生成伪标签的同时为每张伪标签计算质量分数。
4. 使用真实标注样本与伪标签样本共同训练主检测网络 `PNet`。
5. 在本人改进版本中，`PNet` 训练时会根据伪标签质量分数对弱监督样本动态加权。
6. 最终通过 `PNet` 在测试集上输出伪装目标分割结果。

## 2.3 PNet 训练路线

`PNet` 的训练输入与 `ANet` 不同。它不再接收框提示图像，而是直接使用：

- 全监督样本：真实图像 + 真值掩码 + 边缘

在改进版本中，还额外接收：

- 弱监督样本对应的 `quality_score`

训练路线如下：

1. 从全监督数据集和弱监督数据集分别取一个 batch。
2. 将两部分 batch 在 batch 维拼接。
3. 将拼接后的图像送入 `PNet`。
4. 得到多尺度掩码输出和边界输出。
5. 计算区域损失、边界损失和 UAL 损失。
6. 在改进版本中，根据弱监督样本质量分数和 warmup 系数构造 `sample_weight`。
7. 用 `sample_weight` 对各项损失进行加权平均。

## 3.2 Branch 分支结构

对应源码：

- [code/TrainANet/lib/Network.py](C:\Users\resii\Desktop\Noisy-cod-main-onlycode\code\TrainANet\lib\Network.py:31)

每个 `Branch` 包含：

1. `convnext_base()` 编码器
2. `GCM3()` 多尺度融合与频域分解
3. 低频下采样层 `LL_down3`、`LL_down4`
4. 高频/低频与原特征拼接后的 `ETM` 模块

执行路线：

1. `shared_encoder(x)` 提取四层特征 `x1, x2, x3, x4`

3. 通过 `DWT` 将融合特征分解为 `LL, LH, HL, HH`
4. 高频分量 `HH` 与浅层特征拼接，用于恢复细节
5. 低频分量 `LL` 下采样后与深层特征拼接，用于保持语义稳定性

## 3.3 GCM3 与 DWT 的作用

对应源码：

- [code/TrainANet/lib/Network.py](C:\Users\resii\Desktop\Noisy-cod-main-onlycode\code\TrainANet\lib\Network.py:261)
- [code/TrainANet/lib/Network.py](C:\Users\resii\Desktop\Noisy-cod-main-onlycode\code\TrainANet\lib\Network.py:112)

`GCM3` 的主要作用是：

- 对四层特征做统一通道变换
- 上采样到同一尺度后融合
- 对融合特征执行离散小波分解（DWT）

`DWT` 输出：

- `LL`：低频信息，保留整体语义轮廓
- `LH`、`HL`：中间方向细节
- `HH`：高频信息，保留细粒度边缘和纹理

这一步的本质是：  
**把融合后的特征同时从语义和边界两个角度重新组织，以便后续解码器分别利用。**

## 3.4 GPM 的作用

对应源码：

- [code/TrainANet/lib/Network.py](C:\Users\resii\Desktop\Noisy-cod-main-onlycode\code\TrainANet\lib\Network.py:295)

`GPM` 用最高层特征 `x4` 生成 `prior_cam`，即粗粒度先验图。  
这个先验图可以理解为：

- 当前样本中“目标大概在哪儿”
- 哪些区域更值得解码器重点关注

它先通过多分支空洞卷积增强感受野，再输出 1 通道先验图。

## 3.5 解码器 UNetDecoderWithEdges

对应源码：

- [code/TrainANet/lib/Network.py](C:\Users\resii\Desktop\Noisy-cod-main-onlycode\code\TrainANet\lib\Network.py:225)

解码器由四个 `UNetDecoderWithEdgesBlock` 组成，每一级都做两件事：

1. 恢复当前层的分割结果 `y`
2. 生成当前层的边界结果 `bound`

其中的核心逻辑是：

- 先将 `prior_cam` 上采样到当前层尺寸
- 再与当前特征拼接融合
- 利用 `r_prior_cam = 1 - sigmoid(prior_cam)` 对特征进行门控
- 通过 `out_y` 输出前景掩码
- 通过 `out_B` 输出边界图

这说明该解码器不是简单恢复空间分辨率，而是：

- 用先验图指导每层特征恢复
- 用边界分支强化对伪装目标轮廓的学习

## 6.1 改动一：在 ANet 伪标签生成阶段加入质量评分

### 修改前

原始 `TrainANet/Test.py` 的逻辑是：

```python
for j in range(len(res)):
    pre = cv2.resize(res[j], dsize=(H[j].item(), W[j].item()))
    ed = cv2.resize(edge[j], dsize=(H[j].item(), W[j].item()))
    cv2.imwrite(save_path + '/mask' +'/'+name[j].replace(".jpg", ".png"), pre*255.)
    cv2.imwrite(save_path + '/edge' +'/'+name[j].replace(".jpg", ".png"), ed*255.)
```

也就是说，原始代码只做两件事：

- 生成伪标签掩码
- 生成边缘图

但它没有回答一个关键问题：  
**这些伪标签里哪些更可靠，哪些噪声更大？**

### 修改后

现在新增了 `compute_quality_score`：

- [code/TrainANet/Test.py](C:\Users\resii\Desktop\Noisy-cod-main-onlycode\code\TrainANet\Test.py:24)

并在保存伪标签时记录质量分数：

- [code/TrainANet/Test.py](C:\Users\resii\Desktop\Noisy-cod-main-onlycode\code\TrainANet\Test.py:71)
- [code/TrainANet/Test.py](C:\Users\resii\Desktop\Noisy-cod-main-onlycode\code\TrainANet\Test.py:76)

当前关键代码：

```python
score = compute_quality_score(pre, ed)
quality_rows.append({'name': name[j].replace(".jpg", ".png"), 'score': f'{score:.6f}'})
```

### 修改原理

质量评分函数由三部分组成：

1. `mask_conf`
   取前景概率与背景概率较大的那一侧，衡量预测是否“有信心”

2. `edge_conf`
   利用边界响应强度衡量预测轮廓是否清晰

3. `area_penalty`
   判断前景区域占比是否偏离经验合理范围

最终：

```python
score = 0.6 * mask_conf + 0.25 * edge_conf + 0.15 * area_penalty
```

这种设计的直觉是：

- 如果预测区域整体很不确定，分数应该低
- 如果边缘弱、轮廓模糊，分数应该低
- 如果前景面积异常大或异常小，说明伪标签可能不可信，分数也应下降

### 实际意义

这个改动把“伪标签有没有”升级为“伪标签质量高不高”，为后续加权训练提供了样本级依据。

```python
def weighted_mean(loss_vector, sample_weight=None):
    if sample_weight is None:
        return loss_vector.mean()
    sample_weight = sample_weight.view(-1)
    norm_weight = sample_weight / sample_weight.sum().clamp_min(1e-6)
    return torch.sum(loss_vector.view(-1) * norm_weight)
```

### 修改原理

原始代码默认：

> 每个样本同样重要

改进版代码改成：

> 每个样本的重要程度由其质量分数决定

因此必须让损失函数保留到“样本级”再统一做加权平均。

### 实际意义

这一步是“伪标签质量评估”真正作用到训练过程中的核心。

## 6.4 改动四：加入弱监督 warmup 与动态权重系数

### 修改前

原始训练阶段没有弱监督 warmup 机制，也没有样本质量引导：

- 弱监督样本从训练一开始就与全监督样本一起直接参与训练
- 低质量伪标签也会同样影响梯度

### 修改后

新增：

- `get_weak_ratio_coef(epoch, warmup_epoch)`
- `weak_weight`
- `min_weak_weight`
- `weak_warmup_epoch`

对应源码：

- [code/TrainPNet/utils/utils.py](C:\Users\resii\Desktop\Noisy-cod-main-onlycode\code\TrainPNet\utils\utils.py:37)
- [code/TrainPNet/TrainDDP.py](C:\Users\resii\Desktop\Noisy-cod-main-onlycode\code\TrainPNet\TrainDDP.py:136)
- [code/TrainPNet/TrainDDP.py](C:\Users\resii\Desktop\Noisy-cod-main-onlycode\code\TrainPNet\TrainDDP.py:246)

关键逻辑：

```python
weak_ratio_coef = get_weak_ratio_coef(epoch, opt.weak_warmup_epoch)
weak_quality = torch.clamp(quality_weak.to(device), min=opt.min_weak_weight, max=1.0)
sample_weight = torch.cat([
    torch.ones_like(quality_fully, device=device),
    weak_quality * opt.weak_weight * weak_ratio_coef
], dim=0)
```

### 修改原理

这个设计包含两层思想：

1. **样本质量差异**
   高质量伪标签权重大，低质量伪标签权重小

2. **训练阶段差异**
   前期模型尚不稳定，弱监督样本应少参与  
   后期模型已有基础能力，弱监督样本可逐步增加参与度

因此最终权重是：

```text
弱监督样本最终权重 = 质量分数 × 全局弱监督系数 × warmup系数
```

### 实际意义

这相当于给模型一个更稳的学习节奏：

- 先学“可靠的真值信号”
- 再逐渐利用“筛过质量的伪标签”

这样能减弱错误伪标签在训练早期造成的破坏性影响。

## 7. 修改前后模型训练逻辑的本质变化

从更高层次看，修改前后的本质区别是：

### 修改前

- `ANet` 生成伪标签
- `PNet` 直接把所有伪标签当成同样可靠的监督
- 弱监督样本从一开始就大比例参与训练

### 修改后

- `ANet` 不只生成伪标签，还生成每个伪标签的质量分数
- `PNet` 在训练时会读取质量分数
- 每个弱监督样本不再等权，而是根据质量动态赋权
- 同时引入 warmup，使弱监督样本逐步参与训练

可以把这种变化总结为：

> 原始方法：先生成伪标签，再直接用  
> 改进方法：先生成伪标签，再评估质量，最后按质量有区别地使用
