# 毕业设计源码修改计划

## 1. 原始源码基础

本仓库对应 ECCV 2024 论文《Learning Camouflaged Object Detection from Noisy Pseudo Label》，原方法已经包含以下成熟成果，可作为毕业设计的已有基础：

- 两阶段弱监督伪装目标检测框架：先训练 `ANet` 生成伪标签，再训练 `PNet` 完成最终检测。
- 多尺度解码与边缘分支：在 `lib/Network.py` 中已经实现多级预测和边缘辅助监督。
- 不确定性感知损失：训练脚本中已有 `UAL loss` 与多层监督策略。

这些内容可以作为“现有研究基础”或“参考实现依据”写入论文综述与系统设计部分。

## 1.1 与任务书要求的对应关系

根据任务书《基于弱监督学习的伪装目标检测方法设计与实现》，课题的核心要求有两点：

- 探索基于物体标注框的弱监督伪装目标检测方法。
- 重点解决“严重噪声伪标签的矫正”和“模型检测精度保持”问题。

当前源码与任务书的天然对应关系如下：

- `ANet` 使用框提示信息参与训练与预测，符合“基于物体标注框的弱监督方法”要求。
- `PNet` 使用少量真值标注和大量伪标签联合训练，符合“弱监督学习的伪装目标检测方法设计与实现”要求。
- 原论文题目本身即为“从噪声伪标签学习伪装目标检测”，与任务书中“矫正严重噪声的伪标签”高度一致。

因此，本次毕业设计最合理的实现路径不是完全重写模型，而是在原始两阶段框架上，围绕“伪标签噪声抑制”和“检测性能稳定提升”做针对性改进。

## 2. 适合作为本人工作的改动

为保证改动量适中，同时能体现个人工作，本次修改聚焦三点：

1. 伪标签质量评估
   - 在 `ANet` 生成伪标签阶段，为每张伪标签计算质量分数。
   - 质量分数由区域置信度、边缘响应和前景面积约束共同构成。
   - 输出 `quality_scores.csv`，为后续训练提供样本可靠性依据。

2. 弱监督样本动态加权
   - 在 `PNet` 训练阶段引入伪标签质量权重。
   - 高质量伪标签参与更多监督，低质量伪标签被自动降权。
   - 增加 warmup 机制，使模型前期更多依赖真值监督，后期逐渐利用伪标签。

3. 实验复现与工程可用性增强
   - 修复预训练权重的硬编码绝对路径，改为相对路径加载。
   - 将伪标签阶段的 Linux `cp -r` 改为跨平台 Python 复制方式。
   - 训练日志中增加弱监督权重系数与伪标签平均质量，便于实验分析。

## 2.1 为什么这些改动适合写成毕业设计工作

这些改动既不属于简单改参数，也没有脱离原始课题方向，比较适合作为本科毕业设计中的“本人工作”：

- 伪标签质量评估：直接回应任务书中“如何矫正严重噪声伪标签”的问题。
- 动态加权训练：直接回应“在噪声存在时如何保证检测精度”的问题。
- 复现与实验流程优化：保证算法可以完整训练、测试和分析，支撑任务书中的“算法实现”和“实验验证”。

## 3. 论文中可写的创新表述

可以将本人的工作概括为：

- 提出一种面向伪标签噪声抑制的质量评估策略，用于衡量弱监督样本的可信度。
- 设计一种基于质量分数的动态加权训练方法，缓解低质量伪标签对主干检测网络的干扰。
- 在原始两阶段框架基础上实现可复现实验流程优化，提高方法的工程稳定性与可迁移性。

如果在论文中需要更稳妥一些的表述，可以写为：

- 在现有弱监督伪装目标检测框架基础上，引入伪标签质量评估机制。
- 针对伪标签噪声问题，设计样本级动态加权训练策略。
- 完成算法实现、实验复现与结果分析，并验证改进策略的有效性。

## 5. 建议的实验对比

建议至少做三组实验：

1. 原始 PNet 基线。
2. 原始 PNet + 伪标签质量筛选。
3. 原始 PNet + 伪标签质量筛选 + 动态加权训练。

如果时间允许，可补充不同 `ration`（如 `1%`、`5%`、`10%`）下的对比结果，用于支撑“弱标注比例越低时，质量加权越有效”的结论。
